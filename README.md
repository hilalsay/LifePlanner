# Life Planner

A personal productivity web app for planning, habit tracking, mood logging, reading lists, and AI-powered goal coaching. Built with FastAPI, React, and PostgreSQL.

## Features

- **Planning hierarchy** ‚Äî yearly vision, monthly focuses, weekly priorities, daily tasks
- **Calendar views** ‚Äî year, month, week, and day views with task overview
- **Habit tracker** ‚Äî define daily/weekly habits and log completions
- **Mood & health** ‚Äî log daily mood, energy, sleep, and water intake
- **Reading list** ‚Äî track books with Google Books search (covers auto-filled), star ratings, written reviews, and progress pages
- **AI assistant** ‚Äî conversational goal coach that suggests tasks, habits, and priorities; dispatches to Gemini or a local Ollama model
- **Weekly AI review** ‚Äî automatic narrative summary of your week generated by Claude
- **Settings** ‚Äî avatar upload, language toggle (English / Turkish), hide-completed preference
- **Auth** ‚Äî email/password registration, Google OAuth, GitHub OAuth; JWT via httponly cookies

## Tech stack

| Layer | Tech |
|---|---|
| Backend | FastAPI, SQLAlchemy 2, Alembic, Pydantic v2, python-jose, passlib |
| Database | PostgreSQL 16 |
| Frontend | React 18, TypeScript, Vite, Tailwind CSS, shadcn/ui, react-router-dom |

## Prerequisites

- Python 3.12
- Node.js 18+ and npm
- PostgreSQL 16 (or Docker for the DB only)

## Quick start

```bash
# 1. Clone
git clone <repo-url>
cd LifePlanner

# 2. Create your environment file
cp backend/.env.example backend/.env
# Edit backend/.env ‚Äî at minimum add GEMINI_API_KEY (see Environment variables below)

# 3. Install everything and set up the database
./setup.sh

# 4. Start
./start.sh
```

`setup.sh` installs system packages (apt), creates a Python venv, installs Python and npm dependencies, creates the `lifeplanner` PostgreSQL user and `life_planner` database, and runs Alembic migrations.

`start.sh` starts PostgreSQL, runs any new migrations, starts the backend on port 8000, starts the frontend on port 5173, and monitors both processes. Press **Ctrl+C** to stop everything.

Open **http://localhost:5173** in your browser.

The interactive API docs are available at **http://localhost:8000/docs**.

## Using Docker for the database

If you prefer Docker instead of a local PostgreSQL install:

```bash
docker compose up -d   # starts postgres:16-alpine on port 5432
```

Then run `./setup.sh` (it will skip the PostgreSQL install step if the port is already open) and `./start.sh` as normal.

## Environment variables

Create `backend/.env` (copy from `backend/.env.example`).

### Required

| Variable | Description |
|---|---|
| `DATABASE_URL` | PostgreSQL DSN. Default: `postgresql://lifeplanner:lifeplanner123@localhost:5432/life_planner` |

### AI provider

| Variable | Default | Description |
|---|---|---|
| `AI_PROVIDER` | `gemini` | `gemini` or `ollama` |
| `GEMINI_API_KEY` | ‚Äî | Required when `AI_PROVIDER=gemini`. Get one at [aistudio.google.com](https://aistudio.google.com) |
| `ANTHROPIC_API_KEY` | ‚Äî | Optional. Enables AI weekly reviews (uses `claude-sonnet-4-6`) and daily motivational messages |
| `OLLAMA_BASE_URL` | ‚Äî | Required when `AI_PROVIDER=ollama`. E.g. `http://localhost:11434` |
| `OLLAMA_MODEL` | `qwen2.5:3b` | Model to use with Ollama |
| `OLLAMA_HOST_HEADER` | `127.0.0.1` | Ollama's DNS-rebind guard; leave as-is for local installs |

### Security (change in production)

| Variable | Default | Description |
|---|---|---|
| `JWT_SECRET_KEY` | *(placeholder)* | Sign JWTs. Generate with `openssl rand -hex 32` |
| `SESSION_SECRET_KEY` | *(placeholder)* | Sign OAuth session cookies. Use a separate random value |
| `COOKIE_SECURE` | `false` | Set `true` in production (requires HTTPS) |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | `15` | Access token lifetime |
| `REFRESH_TOKEN_EXPIRE_DAYS` | `7` | Refresh token lifetime |

### OAuth (optional)

| Variable | Description |
|---|---|
| `GOOGLE_CLIENT_ID` | Google OAuth app client ID |
| `GOOGLE_CLIENT_SECRET` | Google OAuth app client secret |
| `GITHUB_CLIENT_ID` | GitHub OAuth app client ID |
| `GITHUB_CLIENT_SECRET` | GitHub OAuth app client secret |

For Google OAuth, set the authorized redirect URI to `http://localhost:8000/api/v1/auth/google/callback`.
For GitHub OAuth, set the callback URL to `http://localhost:8000/api/v1/auth/github/callback`.

### URLs

| Variable | Default | Description |
|---|---|---|
| `FRONTEND_URL` | `http://localhost:5173` | Added to CORS allowed origins; set to your production domain |
| `BACKEND_URL` | `http://localhost:8000` | Used in OAuth redirect URIs |

## AI provider configuration

### Gemini (default)

```env
AI_PROVIDER=gemini
GEMINI_API_KEY=AIza...
```

The assistant uses `gemini-2.5-flash-lite`. The free tier is sufficient for personal use.

### Ollama (local / offline)

```bash
# Install Ollama, then pull a model
ollama pull qwen2.5:3b
```

```env
AI_PROVIDER=ollama
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=qwen2.5:3b
```

Any instruction-following model works. Larger models give better goal suggestions; `qwen2.5:3b` runs well on CPU.

## Production deployment

Before exposing the app publicly:

1. **Generate strong secrets**

   ```bash
   openssl rand -hex 32   # use for JWT_SECRET_KEY
   openssl rand -hex 32   # use for SESSION_SECRET_KEY
   ```

2. **Enable secure cookies** ‚Äî set `COOKIE_SECURE=true` and serve the app over HTTPS.

3. **Set the database password** ‚Äî change the default `lifeplanner123` in `docker-compose.yml` and `DATABASE_URL`.

4. **Set `FRONTEND_URL`** to your production domain so CORS is correct.

5. **Run the backend with a process manager** (e.g. gunicorn + uvicorn workers, systemd, or a container) instead of the dev `--reload` server.

## API

All endpoints are prefixed `/api/v1`. Key groups:

| Prefix | Description |
|---|---|
| `/auth` | Register, login, logout, OAuth (Google, GitHub), avatar upload |
| `/planning` | Monthly focuses, weekly priorities, daily tasks |
| `/habits` | Habit definitions and completion logs |
| `/tracking` | Mood, health, books, weekly reviews |
| `/ai` | Chat assistant, natural-language task parsing |

Interactive docs: `http://localhost:8000/docs`
